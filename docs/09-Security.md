# Безопасность

## 1. Работа с файлами
- Проверять пути и права доступа (чтение/запись).
- Нормализовать путь (использовать абсолютные пути).
- Ограничивать размер входных файлов (50 MB).
- Для архивных форматов (DOCX/ODT) проверять безопасность архива.

## 2. Обработка содержимого
- Не исполнять контент файлов.
- Защита от zip-bomb (ограничение суммарного размера и коэффициента сжатия).
- Не отправлять системные пути и метаданные в LLM.

## 3. Сеть
- Таймауты и отмена запросов.
- Обрабатывать ошибки и не утекать в UI стектрейс.
- Разрешать только адрес Ollama, заданный пользователем.

## 4. Логи и приватность
- Логи не должны содержать личные данные.
- Опциональная возможность отключить логирование.
 - Уровень логирования настраивается в UI и хранится в настройках.
