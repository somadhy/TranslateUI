# Интеграция с Ollama

## 1. Базовые конечные точки
- `GET /api/tags` — список моделей.
- `POST /api/show` — детали модели.
- `POST /api/pull` — скачивание модели.
- `POST /api/generate` — генерация перевода.

Базовый адрес берется из настроек (по умолчанию `http://localhost:11434`).

## 2. Проверка доступности
- При старте приложения выполнить “health check” через `GET /api/tags`.
- При ошибке показывать сообщение о недоступности Ollama и предложить проверить URL.

## 3. Проверка наличия модели
- Получить список через `/api/tags`.
- Если выбранная модель отсутствует:
  - Показывать предупреждение.
  - Предлагать “Скачать”.

## 4. Скачивание модели
- Запуск через `/api/pull`.
- Отображение прогресса (если доступен).
- В случае ошибки — понятное сообщение.

## 5. Vision поддержка
- Проверять метаданные `/api/show`.
- Если метаданные указывают поддержку изображений — активировать режим “Изображения”.
- Если нет — скрыть/заблокировать режим.

## 6. Запрос перевода
**POST** `/api/generate`:
- `model`: выбранная модель.
- `prompt`: строка промпта.
- `stream`: `false` (упрощает UI).
- `images`: массив base64 изображений (только для vision).

## 7. Таймауты и отмена
- Таймауты: 30-120 секунд (настраиваемо).
- Поддержка `CancellationToken` для остановки перевода.
