# Тестирование

## 1. Unit-тесты
- `PromptBuilder`: корректность формата и двух пустых строк.
- `LanguageCatalog`: поиск и сопоставление кодов.
- `SettingsService`: загрузка/сохранение, валидация.
- `FileHandlers`: чтение/запись базовых форматов.

## 2. Интеграционные тесты
- `OllamaClient` с `HttpMessageHandler` mock.
- Проверка обработки ошибок сети/таймаутов.
- Проверка сценариев “модель отсутствует/скачивается”.
 - Проверка смены уровня логирования из настроек.

## 3. UI-тесты
- Avalonia Headless (если возможно).
- Сценарии: перевод текста, выбор файла, изменение настроек.

## 4. Тестовые данные
- Набор образцов для всех поддерживаемых форматов.
- Небольшие и большие файлы для проверки производительности.
