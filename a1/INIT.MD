# Общее описаниние
Относительно минималистичное Desktop приложение - переводчик текста.
Оно должно будет иметь доступ к файловой системе и API Olama. 
Должено быть достаточно эстетично, но строго, но при этом не слишком ресурсоёмко (RAM, Диск, CPU)

# Перевод 
- Перевод текста - два окна с исходным текстом и результат. Предлагать сохранить перевод.
- Перевод файлов - выбор файла с испоьзованием диалога или перетаскивание. Выбор пути куда положить результат. Ссылка на файл результата что бы сразу открыть.
- Перевод с изображения. Доступен только если модель LLM поддерживанеи изображения
- Должна быть возможность выбота исходного и целевого языка
- Должна быть возможность выбрать LLM модель
- Перевод не по мере набора текста а по нажатию кнопки "Перевод"

# Файлы 
Поддерживать работу с файлами .PDF, .TXT, .MD, .DOCX, .ODT, PNG, JPEG, TIFF (реализовать в соответствии с паттернами которые позволят легко добавлять поддержку новых файлов)

# Настройки
- Исходный и целевой язык по умолчанию
- Адрес Olama сервиса
- Выбор модели в виде по умолчанию
- Выбор языка интерфейса (по умолчанию English)

# Выбор модели
- Должен быть реализован в виде ползунка с дискретными положениями соответствующими размеру моделей 
(translategemma:4b,translategemma:12b,translategemma:27b) и отметками от левой "Скорость" до "Точность" 
- Если модель ещё не скачана Olama должно подсвечитавться предупреждение что потребуется время на скачку.
- При выборе модели которой ещё нет в Olama предлагать кнопку для её скачивания сразу или скачивать автоматически при инициировании перевода.

# Интерфейс
- Все текстовые ресурсы должны быть отдельно и подключаться в зависимости от языка интерфейса, сразу должны быть доткпны ресурсы на Русском и English
- Возможность менять размер окна и убирать окно в tray
- При изменении размера интерфейс должен быть адаптивным
- Полноценная работа с буффером обмена
- Строгая эстетика
